      55                 :            : /** A key from a CWallet's keypool
      56                 :            :  *
      57                 :            :  * The wallet holds one (for pre HD-split wallets) or several keypools. These
      58                 :            :  * are sets of keys that have not yet been used to provide addresses or receive
      59                 :            :  * change.
      60                 :            :  *
      61                 :            :  * The Bitcoin Core wallet was originally a collection of unrelated private
      62                 :            :  * keys with their associated addresses. If a non-HD wallet generated a
      63                 :            :  * key/address, gave that address out and then restored a backup from before
      64                 :            :  * that key's generation, then any funds sent to that address would be
      65                 :            :  * lost definitively.
      66                 :            :  *
      67                 :            :  * The keypool was implemented to avoid this scenario (commit: 10384941). The
      68                 :            :  * wallet would generate a set of keys (100 by default). When a new public key
      69                 :            :  * was required, either to give out as an address or to use in a change output,
      70                 :            :  * it would be drawn from the keypool. The keypool would then be topped up to
      71                 :            :  * maintain 100 keys. This ensured that as long as the wallet hadn't used more
      72                 :            :  * than 100 keys since the previous backup, all funds would be safe, since a
      73                 :            :  * restored wallet would be able to scan for all owned addresses.
      74                 :            :  *
      75                 :            :  * A keypool also allowed encrypted wallets to give out addresses without
      76                 :            :  * having to be decrypted to generate a new private key.
      77                 :            :  *
      78                 :            :  * With the introduction of HD wallets (commit: f1902510), the keypool
      79                 :            :  * essentially became an address look-ahead pool. Restoring old backups can no
      80                 :            :  * longer definitively lose funds as long as the addresses used were from the
      81                 :            :  * wallet's HD seed (since all private keys can be rederived from the seed).
      82                 :            :  * However, if many addresses were used since the backup, then the wallet may
      83                 :            :  * not know how far ahead in the HD chain to look for its addresses. The
      84                 :            :  * keypool is used to implement a 'gap limit'. The keypool maintains a set of
      85                 :            :  * keys (by default 1000) ahead of the last used key and scans for the
      86                 :            :  * addresses of those keys.  This avoids the risk of not seeing transactions
      87                 :            :  * involving the wallet's addresses, or of re-using the same address.
      88                 :            :  * In the unlikely case where none of the addresses in the `gap limit` are
      89                 :            :  * used on-chain, the look-ahead will not be incremented to keep
      90                 :            :  * a constant size and addresses beyond this range will not be detected by an
      91                 :            :  * old backup. For this reason, it is not recommended to decrease keypool size
      92                 :            :  * lower than default value.
      93                 :            :  *
      94                 :            :  * The HD-split wallet feature added a second keypool (commit: 02592f4c). There
      95                 :            :  * is an external keypool (for addresses to hand out) and an internal keypool
      96                 :            :  * (for change addresses).
      97                 :            :  *

      99                 :            :  * stored as sets of indexes in the wallet (setInternalKeyPool,
